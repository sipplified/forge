Forge uses your Kubernetes manifest and Dockerfile -- so it’s easy to use and learn.

Dynamically configure your Kubernetes manifests with Jinja2 templates

Define service dependencies, so you can deploy a multi-service application with one command

Native integration with Ambassador lets you easily do canary deployments

Forge is a single binary that runs anywhere: in your CI system, on your laptop, or anywhere else.

Are you a Kubernetes application developer who wants to deploy your application from source into Kubernetes? Tired of waiting for your CD pipeline to run just so you can build and run one test?

With forge, you:

  1. Define how your application is built in a Dockerfile, and how it’s deployed in a Kubernetes manifest.

  2. Optionally define any dependencies and runtime configuration in a service.yaml file.

  3. Run forge deploy which will deploy all your services into Kubernetes.

Get started in a few minutes:

  1. Install Forge

  2. Configure Forge

    forge setup

  3. Add a Dockerfile, Kubernetes manifest, and service.yaml to your source repo

    git clone https://github.com/datawire/hello-forge.git

  4. Deploy your service

    forge deploy
